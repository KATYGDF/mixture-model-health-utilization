1 · Objetivo

A página de Análises apresenta diretamente o modelo final: uma Mistura de Binomiais Negativas ajustada por um algoritmo EM implementado do zero. Para um leitor que encontra esse modelo pela primeira vez, a impressão pode ser a de um algoritmo complexo construído arbitrariamente.

O objetivo desta seção é mostrar que esse modelo não foi arbitrário — ele é a consequência natural de uma série de escolhas motivadas pelas limitações dos modelos anteriores. A narrativa segue uma trajetória progressiva:

1

GLM Poisson — Equidispersão

Assume \(\mathrm{Var}[Y] = \mathbb{E}[Y]\). Dados de utilização violam isso sistematicamente. A Poisson subestima a variabilidade e produz resíduos estruturados.

2

GLM Binomial Negativa — Sobredispersão

Corrige a equidispersão via parâmetro \(\theta\). Mas pressupõe uma única população. Quando existem subgrupos com padrões distintos, a NB global está errada por construção.

3

Mistura Gaussiana (mixtools) — Suporte errado

Introduz a ideia de múltiplos componentes via EM. Mas aplica distribuições Normais a dados discretos: \(P(Y < 0) > 0\) é matematicamente impossível para contagens.

4

Mistura Poisson (flexmix) — Equidispersão intra-componente

Usa a família correta (discreta, não-negativa), mas força equidispersão dentro de cada componente. Dados de saúde suplementar violam isso mesmo dentro de cada subgrupo.

5

Mistura NB univariada (flexmix) — Marginal apenas

Família e dispersão corretas, mas opera variável por variável. A estrutura de agrupamento emerge da combinação das variáveis; modelar cada uma separadamente perde a dependência conjunta.

6

Mistura NB multivariada — EM customizado ✓

Cada componente é um produto de NB independentes por variável, com \((\mu_{hj}, \theta_{hj})\) estimados conjuntamente via E-step / M-step. Inicialização K-means, múltiplos reinícios, seleção por BIC.

2 · Parte I — Sobredispersão: por que a Poisson falha

A hipótese de equidispersão

A distribuição de Poisson para uma variável de contagem \(Y\) presupõe que variância e média coincidem:

$$\mathrm{Var}[Y] = \mathbb{E}[Y]$$

Essa hipótese é violada em praticamente todos os dados de utilização de serviços de saúde. A heterogeneidade entre beneficiários — diferenças de idade, cronicidade, perfil de risco — produz variâncias muito superiores às médias. A razão \(\mathrm{Var}[Y] / \mathbb{E}[Y]\), chamada índice de sobredispersão, serve como diagnóstico: 

# Razão variância/média por variável (V3)
vars <- c("consultas", "ps", "exames", "internacoes", "terapias")

stats <- data.frame(
  variavel  = vars,
  media     = sapply(vars, function(v) round(mean(df[[v]]), 2)),
  variancia = sapply(vars, function(v) round(var(df[[v]]),  2)),
  zeros_pct = sapply(vars, function(v) round(100 * mean(df[[v]] == 0), 1))
)
stats$razao_v_m <- round(stats$variancia / stats$media, 1)
print(stats)

A tabela abaixo sintetiza o padrão esperado nos dados simulados:

Variável Média Variância Var/Média % zeros Interpretação
consultas ≈ 14≈ 120 ≈ 8,6≈ 0% sobredispersão moderada
ps ≈ 3≈ 18 ≈ 6,0≈ 5% sobredispersão moderada
exames ≈ 12≈ 250 ≈ 20,8≈ 2% sobredispersão elevada
internacoes ≈ 0,6≈ 18 ≈ 30,0≈ 55% excesso de zeros + cauda longa
terapias ≈ 2≈ 70 ≈ 35,0≈ 40% excesso de zeros + cauda longa
Valores representativos para \(n = 5{,}000\) com \(G = 4\) grupos simulados. Execute construcao_progressiva.R para os valores exatos. Para a Poisson ser adequada, todas as razões deveriam ser ≈ 1.

Comparação Poisson vs Binomial Negativa — GLM univariado

Para tornar a falha da Poisson concreta, ajusta-se um GLM univariado a exames — a variável com sobredispersão mais pronunciada — com ambas as famílias: 

# GLM Poisson vs Binomial Negativa (exames)
library(MASS)

fit_pois <- glm(exames ~ 1, data = df, family = poisson)
fit_nb   <- MASS::glm.nb(exames ~ 1, data = df)

cat("AIC Poisson:", round(AIC(fit_pois), 1), "\n")
cat("AIC NB:     ", round(AIC(fit_nb),   1), "\n")
cat("theta NB:   ", round(fit_nb$theta,  4), "\n")
# theta pequeno ⟹ sobredispersão intensa (1/theta = índice de dispersão)

A diferença de AIC entre os dois modelos é tipicamente da ordem de milhares — evidência de que a equidispersão da Poisson é uma restrição inaceitável para esses dados.

"Count data from insurance claims are almost invariably overdispersed. Poisson regression systematically underestimates variance, producing anticonservative inference and inflated test statistics." — Ahmad et al. (2025, p. 192)

3 · Parte II — A Binomial Negativa ainda não basta

Sobredispersão global vs heterogeneidade latente

A Binomial Negativa resolve a equidispersão adicionando um parâmetro de dispersão \(\theta > 0\). A distribuição marginal de \(Y\) torna-se:

$$P(Y = y \mid \mu, \theta) = \binom{y + \theta - 1}{y} \left(\frac{\mu}{\mu + \theta}\right)^{y} \left(\frac{\theta}{\mu + \theta}\right)^{\theta}$$

com \(\mathbb{E}[Y] = \mu\) e \(\mathrm{Var}[Y] = \mu + \mu^2/\theta\). Quando \(\theta \to \infty\), recupera-se a Poisson; quando \(\theta\) é pequeno, a cauda direita é pesada.

O problema é que um GLM NB pressupõe uma única população homogênea: os parâmetros \((\mu, \theta)\) são globais. Quando a população é composta por subgrupos com perfis de utilização qualitativamente distintos — beneficiários jovens e saudáveis, adultos com doenças crônicas, idosos de alta complexidade, pacientes em regime de terapias intensivas — a NB global está produzindo médias e dispersões que não descrevem nenhum dos subgrupos com fidelidade. 

# Diagnóstico: resíduos de Pearson do modelo NB
resid_nb <- residuals(fit_nb, type = "pearson")

# Curtose empírica dos resíduos
# Normal ⟹ curtose ≈ 3; valores elevados indicam estrutura residual
kurt <- mean((resid_nb - mean(resid_nb))^4) / var(resid_nb)^2
cat(sprintf("Curtose dos resíduos NB: %.2f\n", kurt))

# Histograma dos resíduos revela bimodalidade
# hist(resid_nb, breaks = 60, main = "Resíduos Pearson — NB global")

Resíduos de Pearson com distribuição bimodal — ou com curtose muito superior a 3 — sinalizam que o modelo está misturando observações de subpopulações distintas em um único componente. A Binomial Negativa captura o excesso de dispersão, mas não explica de onde ele vem.

Analogia: a NB é como calcular a média de peso de homens e mulheres adultos juntos — obtém-se um número válido, mas que não representa bem nenhum dos dois grupos. Um modelo de mistura resolve isso atribuindo cada observação a um componente.

4 · Parte III — Primeiros modelos de mistura: mixtools

A ideia central dos mixture models

Um modelo de mistura finita supõe que os dados foram gerados por \(g\) subpopulações latentes com pesos \(\pi_1, \ldots, \pi_g\) e densidades \(f_1, \ldots, f_g\):

$$f(y;\Psi) = \sum_{h=1}^{g} \pi_h \, f_h(y;\psi_h), \qquad \sum_{h=1}^{g} \pi_h = 1, \quad \pi_h > 0$$

O pacote mixtools implementa o algoritmo EM para misturas de distribuições normais via normalmixEM(). Como primeiro passo pedagógico, aplica-se essa mistura à transformação logarítmica de exames: \(y^* = \log(1 + y)\). 

# Mistura Gaussiana via mixtools (aplicada a log1p(exames))
library(mixtools)

y_log <- log1p(df$exames)

# Seleção de g por BIC (calculado manualmente a partir do log-lik)
bic_gauss <- numeric(5)   # g = 2..6
for (g in 2:6) {
  set.seed(42)
  fit <- suppressWarnings(
    mixtools::normalmixEM(y_log, k = g, maxit = 500, epsilon = 1e-6)
  )
  k_par        <- 3L * g - 1L   # g médias + g dp + (g-1) pesos
  bic_gauss[g - 1] <- -2 * fit$loglik + k_par * log(length(y_log))
}

g_opt <- which.min(bic_gauss) + 1L
cat("g selecionado:", g_opt, "\n")

# Ajuste final
set.seed(42)
fit_gauss <- suppressWarnings(
  mixtools::normalmixEM(y_log, k = g_opt, maxit = 500, epsilon = 1e-6)
)
cat("Pesos:  ", round(fit_gauss$lambda, 3), "\n")
cat("Médias: ", round(fit_gauss$mu,     3), "(escala log1p)\n")
cat("DPs:    ", round(fit_gauss$sigma,  3), "\n")

Por que a mistura Gaussiana é pedagogicamente útil mas tecnicamente inadequada

A mistura Gaussiana demonstra claramente a mecânica do EM: o E-step calcula a probabilidade posterior de cada observação pertencer a cada componente; o M-step atualiza médias, desvios e pesos. A convergência da log-verossimilhança é monótona e observável.

Contudo, três problemas tornam essa família inadequada para dados de contagem:

Limitação 1 — Suporte contínuo: a Normal tem suporte \(\mathbb{R}\), mas contagens pertencem a \(\mathbb{N}_0 = \{0, 1, 2, \ldots\}\). A função de densidade avalia probabilidades para valores negativos, que são matematicamente impossíveis para a variável real. Em escala original, \(P(Y < 0) > 0\).
Limitação 2 — Massa em zero: variáveis como internacoes e terapias apresentam 40–55% de zeros. Uma Normal, mesmo com média próxima de zero, atribui densidade a valores negativos ao invés de concentrar massa no ponto zero.
Limitação 3 — Assimetria residual: mesmo após \(\log(1+y)\), as distribuições de utilização retêm assimetria à direita, especialmente em cauda longa. Uma Normal em log1p approxima mal esses dados.

5 · Parte IV — Mixturas com famílias de contagem: flexmix

Mistura Poisson

O pacote flexmix permite especificar modelos de mistura com componentes GLM, corrigindo o problema do suporte. A mistura Poisson é o primeiro passo dentro das distribuições discretas: 

# Mistura Poisson via flexmix
library(flexmix)

set.seed(42)
fit_pois_mix <- flexmix::flexmix(
  exames ~ 1, data = df, k = 3,
  model   = flexmix::FLXMRglm(family = "poisson"),
  control = list(iter.max = 300, minprior = 0.02)
)
BIC(fit_pois_mix)
summary(fit_pois_mix)   # tamanho e parâmetros dos componentes

A mistura Poisson resolve os problemas de suporte e de massa em zero. Entretanto, ao assumir equidispersão dentro de cada componente (\(\mathrm{Var}[Y_h] = \mathbb{E}[Y_h]\)), ela tende a fragmentar clusters reais em múltiplos componentes espúrios para absorver a dispersão residual.

Limitação — Equidispersão intra-componente: a Poisson impõe \(\mathrm{Var}[Y_h] = \mu_h\) em cada grupo \(h\). Se beneficiários do mesmo grupo ainda apresentam variabilidade extra, o algoritmo cria componentes adicionais desnecessários — o BIC tende a superestimar \(g\).

Mistura Binomial Negativa

A mistura NB substitui cada componente Poisson por uma NB, permitindo sobredispersão livre dentro de cada grupo: 

# Mistura Binomial Negativa via flexmix
set.seed(42)
fit_nb_mix <- flexmix::flexmix(
  exames ~ 1, data = df, k = 3,
  model   = flexmix::FLXMRnegbin(),
  control = list(iter.max = 300, minprior = 0.02)
)
BIC(fit_nb_mix)

# Parâmetros estimados por componente
parameters(fit_nb_mix)

A mistura NB via flexmix combina família adequada com flexibilidade de dispersão por componente. É um modelo correto do ponto de vista distribuicional — e confirma a estrutura de múltiplos grupos.

6 · Curvas de BIC e seleção de g

O BIC penaliza a log-verossimilhança pelo número de parâmetros livres \(d(g)\) e pelo tamanho amostral \(n\):

$$\mathrm{BIC}(g) = -2\,\hat{\ell}(\hat{\Psi}_g) + d(g)\,\log n$$

Comparar as curvas de BIC ao longo de \(g = 2, \ldots, 6\) para os três modelos de mistura revela como cada família responde ao critério de seleção: 

# BIC para g = 2..6 em três famílias (exames, univariado)
bic_gauss   <- numeric(5)
bic_pois_fx <- numeric(5)
bic_nb_fx   <- numeric(5)

for (g in 2:6) {
  set.seed(42)
  # Gaussiana (log1p)
  fit_g <- suppressWarnings(
    mixtools::normalmixEM(log1p(df$exames), k = g, maxit = 500, epsilon = 1e-6)
  )
  bic_gauss[g-1] <- -2 * fit_g$loglik + (3L*g - 1L) * log(nrow(df))

  # Poisson
  mod_p <- flexmix::flexmix(exames ~ 1, data = df, k = g,
    model = flexmix::FLXMRglm(family = "poisson"),
    control = list(iter.max = 300, minprior = 0.02))
  bic_pois_fx[g-1] <- BIC(mod_p)

  # NB
  mod_nb <- tryCatch(
    flexmix::flexmix(exames ~ 1, data = df, k = g,
      model = flexmix::FLXMRnegbin(),
      control = list(iter.max = 300, minprior = 0.02)),
    error = function(e) NULL)
  if (!is.null(mod_nb)) bic_nb_fx[g-1] <- BIC(mod_nb)
}

# Curvas (opcional: ggplot2)
bic_df <- data.frame(
  g      = rep(2:6, 3),
  BIC    = c(bic_gauss, bic_pois_fx, bic_nb_fx),
  modelo = rep(c("Gaussiana", "Poisson", "NB"), each = 5)
)

Interpretação esperada das curvas:

  • Gaussiana (log1p): mínimo em \(g \approx 3\) ou 4. A transformação log1p melhora a simetria, mas a mistura Normal ainda não captura com fidelidade os padrões discretos.
  • Poisson: mínimo em \(g \geq 4\), frequentemente em 5 ou 6, pois componentes extras são criados para absorver a sobredispersão residual.
  • NB: mínimo em \(g = 3\) ou 4 para a variável exames isolada. Para o modelo multivariado em V3 (EM customizado), o BIC seleciona corretamente \(g = 4\) — o número verdadeiro de grupos.
As curvas acima são para o modelo univariado em exames. O EM customizado da página de Análises opera sobre \(\mathbf{Y} = (\texttt{consultas}, \texttt{ps}, \texttt{exames}, \texttt{internacoes}, \texttt{terapias})\) em conjunto — a informação multivariada é que permite ao BIC convergir para \(g = 4\) de forma estável.

7 · Probabilidades posteriores

Uma das vantagens centrais dos modelos de mistura em relação ao K-means é que o pertencimento a um grupo não é binário: cada observação recebe uma probabilidade posterior \(\hat{\tau}_{ih}\) de pertencer ao componente \(h\):

$$\hat{\tau}_{ih} = \frac{\hat{\pi}_h \, f_h(y_i;\hat{\psi}_h)} {\sum_{k=1}^{g} \hat{\pi}_k \, f_k(y_i;\hat{\psi}_k)}$$ 
# Probabilidades posteriores do modelo NB (flexmix)
post     <- flexmix::posterior(fit_nb_mix)   # matriz n × g
prob_max <- apply(post, 1, max)              # prob. máxima por observação

cat(sprintf("Média das prob. máximas:   %.3f\n", mean(prob_max)))
cat(sprintf("Mediana das prob. máximas: %.3f\n", median(prob_max)))
cat(sprintf("Proporção > 0,90:          %.1f%%\n", 100 * mean(prob_max > 0.9)))

Três padrões são informativos:

Padrão Probabilidade máxima Interpretação
Classificação clara > 0,95 Beneficiário bem separado do grupo adjacente
Região híbrida 0,60 – 0,85 Perfil intermediário entre dois componentes
Alta ambiguidade ≈ 0,50 Sobreposição acentuada; grupos mal separados

Probabilidades intermediárias não representam falha do modelo — representam heterogeneidade contínua real. Em contextos de auditoria de saúde suplementar, beneficiários na região híbrida são especialmente relevantes: padrões suspeitos raramente exibem separação perfeita.

8 · Limitações e necessidade do EM customizado

O que a mistura NB univariada com flexmix não faz

A mistura NB via flexmix é um passo correto e importante. Contudo, ela opera em uma variável de cada vez. A estrutura de agrupamento em saúde suplementar emerge da combinação de variáveis: um beneficiário com alta utilização de exames e terapias simultaneamente pertence a um grupo diferente de quem tem alta utilização apenas de exames.

Para capturar essa estrutura multivariada, é necessário modelar a distribuição conjunta \(\mathbf{Y} = (Y_1, \ldots, Y_p)\) dentro de cada componente. Sob a hipótese de independência condicional — razoável após condicionar no grupo latente —, o E-step opera sobre o produto de densidades NB:

$$f_h(\mathbf{y};\psi_h) = \prod_{j=1}^{p} \mathrm{NB}\!\left(y_j;\, \mu_{hj},\, \theta_{hj}\right)$$

Isso exige estimar \(2gp\) parâmetros de componente (médias e dispersões) mais \(g-1\) pesos, totalizando \(d(g) = 2gp + (g-1)\) parâmetros. Para \(g = 4\) e \(p = 5\) (V3), isso resulta em \(d(4) = 43\) parâmetros.

Limitações específicas do flexmix para este problema

Aspecto flexmix EM customizado
Escopo univariado por componente ou multivariado via fórmula cbind produto de NB independentes por variável, multivariado nativo
Inicialização aleatória (padrão) K-means em log1p(Y) + múltiplos reinícios aleatórios
Atualização de θ otimização numérica interna otimização em log-espaço com log(θ) para garantir θ > 0
Controle de convergência iter.max, minprior tol na variação da log-verossimilhança entre iterações
Múltiplos reinícios via loop externo nativo (n_init = 8), melhor fit selecionado pelo BIC
Critério de seleção de g BIC via BIC(mod) BIC calculado como −2·ℓ̂ + d(g)·log(n)
O EM customizado não é metodologicamente superior ao flexmix em todos os contextos — é adequado especificamente para este problema: mistura NB multivariada com variáveis de contagem de saúde suplementar, onde o controle fino sobre inicialização e convergência é determinante para a estabilidade do BIC.

Com essa motivação estabelecida, a página de Análises apresenta o EM customizado aplicado sistematicamente a cinco conjuntos de variáveis (V1–V5), comparando com K-means como baseline geométrico livre de distribuição.

9 · Código R

Script completo: src/03_analises/construcao_progressiva.R 

# =============================================================================
# construcao_progressiva.R
# Evolução metodológica: da Poisson à Mistura NB customizada
# V3 = consultas + ps + exames + internacoes + terapias
# =============================================================================


# ── 1. Parâmetros ─────────────────────────────────────────────────────────────

CAMINHO_CSV <- "../../dados/dados_simulados.csv"
SEED        <- 42L
G_MAX       <- 6L
VARS_V3     <- c("consultas", "ps", "exames", "internacoes", "terapias")
DIR_RDS     <- "rds"


# ── 2. Pacotes ────────────────────────────────────────────────────────────────

pkgs  <- c("MASS", "mixtools", "flexmix")
novos <- pkgs[!sapply(pkgs, requireNamespace, quietly = TRUE)]
if (length(novos) > 0) install.packages(novos, repos = "https://cloud.r-project.org")
suppressPackageStartupMessages({
  library(MASS); library(mixtools); library(flexmix)
})


# ── 3. Dados ──────────────────────────────────────────────────────────────────

df_raw <- read.csv(CAMINHO_CSV, stringsAsFactors = FALSE, encoding = "UTF-8")
candidatos <- c("grupo_latente", "grupo", "grupo_real", "grupo_idx")
col_grupo   <- candidatos[candidatos %in% names(df_raw)][1]
df <- df_raw
names(df)[names(df) == col_grupo] <- "grupo_real"
df$grupo_real <- factor(df$grupo_real)

cat(sprintf("\nDados: %d obs · %d grupos reais · V3\n", nrow(df), nlevels(df$grupo_real)))


# ── PARTE I: Sobredispersão ───────────────────────────────────────────────────

stats_mv <- data.frame(
  variavel  = VARS_V3,
  media     = sapply(VARS_V3, function(v) round(mean(df[[v]]), 2)),
  variancia = sapply(VARS_V3, function(v) round(var(df[[v]]),  2)),
  zeros_pct = sapply(VARS_V3, function(v) round(100 * mean(df[[v]] == 0), 1))
)
stats_mv$razao_v_m <- round(stats_mv$variancia / stats_mv$media, 1)
print(stats_mv)

y_exames <- df$exames
fit_pois  <- glm(y_exames ~ 1, family = poisson)
fit_nb    <- MASS::glm.nb(y_exames ~ 1)
cat(sprintf("AIC Poisson: %.1f | AIC NB: %.1f\n", AIC(fit_pois), AIC(fit_nb)))


# ── PARTE II: Heterogeneidade latente ─────────────────────────────────────────

resid_nb <- residuals(fit_nb, type = "pearson")
kurt_nb  <- mean((resid_nb - mean(resid_nb))^4) / var(resid_nb)^2
cat(sprintf("Curtose resíduos NB: %.2f\n", kurt_nb))


# ── PARTE III: mixtools — mistura Gaussiana em log1p ─────────────────────────

y_log     <- log1p(y_exames)
bic_gauss <- numeric(G_MAX - 1)
for (g in 2:G_MAX) {
  set.seed(SEED)
  fit_tmp        <- suppressWarnings(
    mixtools::normalmixEM(y_log, k = g, maxit = 500, epsilon = 1e-6))
  k_par          <- 3L * g - 1L
  bic_gauss[g-1] <- -2 * fit_tmp$loglik + k_par * log(length(y_log))
  cat(sprintf("  [g=%d: BIC=%.1f]", g, bic_gauss[g-1]))
}
cat("\n")
g_gauss   <- which.min(bic_gauss) + 1L
set.seed(SEED)
fit_gauss <- suppressWarnings(
  mixtools::normalmixEM(y_log, k = g_gauss, maxit = 500, epsilon = 1e-6))


# ── PARTE IV: flexmix — Poisson e NB ─────────────────────────────────────────

# Mistura Poisson
bic_pois_fx <- sapply(2:G_MAX, function(g) {
  set.seed(SEED)
  mod <- flexmix::flexmix(y_exames ~ 1, k = g,
    model   = flexmix::FLXMRglm(family = "poisson"),
    control = list(iter.max = 300, minprior = 0.02))
  BIC(mod)
})

# Mistura NB
bic_nb_fx  <- numeric(G_MAX - 1)
fits_nb_fx <- vector("list", G_MAX - 1)
for (g in 2:G_MAX) {
  set.seed(SEED)
  mod <- tryCatch(
    flexmix::flexmix(y_exames ~ 1, k = g,
      model   = flexmix::FLXMRnegbin(),
      control = list(iter.max = 300, minprior = 0.02)),
    error = function(e) NULL)
  if (!is.null(mod)) { bic_nb_fx[g-1] <- BIC(mod); fits_nb_fx[[g-1]] <- mod }
  cat(sprintf("  [g=%d: BIC=%.1f]", g, bic_nb_fx[g-1]))
}
cat("\n")
g_nb_fx    <- which.min(bic_nb_fx[bic_nb_fx > 0]) + 1L
mod_nb_best <- fits_nb_fx[[g_nb_fx - 1L]]


# ── PARTE V: Probabilidades posteriores e resumo ──────────────────────────────

post_nb  <- flexmix::posterior(mod_nb_best)
prob_max <- apply(post_nb, 1, max)
cat(sprintf("Prob. máxima — média: %.3f · pct>0,9: %.1f%%\n",
            mean(prob_max), 100 * mean(prob_max > 0.9)))
cat(sprintf("AIC Poisson: %.1f · AIC NB: %.1f\n", AIC(fit_pois), AIC(fit_nb)))
cat(sprintf("BIC Mix Pois (g=%d): %.1f\n", which.min(bic_pois_fx)+1L, min(bic_pois_fx)))
cat(sprintf("BIC Mix NB   (g=%d): %.1f\n", g_nb_fx, min(bic_nb_fx[bic_nb_fx > 0])))
