1 · Objetivo e estrutura

A página Construção Progressiva mostra por que uma Mistura de Binomiais Negativas (NB) é necessária: a Poisson falha por equidispersão; a NB univariada falha por homogeneidade; o flexmix multivariado não permite fixar a parametrização NB₂ nem múltiplos reinícios com critério BIC. A solução é implementar o algoritmo EM do zero.

Esta página acompanha o arquivo comparar_variaveis.R linha por linha, justificando cada decisão de implementação com referência a Ng, Xiang & Yau (2019) — o texto de referência adotado para a modelagem de misturas em dados de saúde.

Convenção de notação: \(n\) = número de beneficiários; \(p\) = número de variáveis (tipos de utilização); \(g\) = número de componentes (grupos latentes); \(h = 1,\ldots,g\) indexa componentes; \(j = 1,\ldots,p\) indexa variáveis; \(i = 1,\ldots,n\) indexa observações. \(Y\) é a matriz \(n \times p\) de contagens observadas.

2 · O modelo de mistura finita

Modelo Definição do modelo de mistura (Ng et al. §1.2, p. 8)
Um modelo de mistura finita representa a densidade conjunta de \(Y_i\) como combinação convexa de \(g\) densidades componentes. Cada beneficiário pertence a exatamente um componente latente, mas esse pertencimento é desconhecido (variável latente \(Z_i\)).
$$f(y_i;\,\Psi) \;=\; \sum_{h=1}^{g} \pi_h\, f_h(y_i;\,\psi_h)$$

onde \(\pi_h > 0\), \(\sum_{h=1}^{g}\pi_h = 1\) são as proporções de mistura e \(f_h\) é a densidade do componente \(h\) com parâmetros \(\psi_h\).

"A finite mixture model with \(g\) components is characterized by the mixture density \(f(y;\Psi) = \sum_{h=1}^{g}\pi_h f_h(y;\psi_h)\), where \(\pi_h > 0\) for \(h = 1,\ldots,g\) and \(\sum_{h=1}^{g}\pi_h = 1\)." Ng, Xiang & Yau (2019) — §1.2, p. 8

No código, o vetor \(\Psi = (\pi_1,\ldots,\pi_g,\,\mu_{11},\ldots,\mu_{gp},\,\theta_{11},\ldots,\theta_{gp})\) é representado pelos objetos pi_h, mu (matriz \(g \times p\)) e theta (matriz \(g \times p\)):

3 · Parametrização NB₂ por componente e variável

NB₂ Distribuição Binomial Negativa — componente e variável (Ng et al. §4.4, p. 81)
Para dados de saúde suplementar, a sobredispersão é a regra: beneficiários mais utilizadores produzem variâncias muito superiores à média. A NB₂ acomoda isso via parâmetro de dispersão \(\theta > 0\): quanto menor \(\theta\), maior o excesso de variância em relação à Poisson.
$$f_{NB}(y;\,\mu,\theta) = \binom{y + \theta - 1}{y} \left(\frac{\theta}{\theta + \mu}\right)^{\!\theta} \left(\frac{\mu}{\theta + \mu}\right)^{\!y}, \quad y = 0,1,2,\ldots$$
$$\mathbb{E}[Y] = \mu, \qquad \mathrm{Var}[Y] = \mu + \dfrac{\mu^2}{\theta}$$
"The NB2 parameterization has variance \(\mu + \mu^2/\theta\), making \(\theta\) directly interpretable as an overdispersion parameter: as \(\theta \to \infty\) the NB approaches the Poisson." Ng, Xiang & Yau (2019) — §4.4, p. 81

Sob independência condicional — dada a componente \(h\), as \(p\) variáveis são independentes — a densidade conjunta do vetor \(y_i = (y_{i1},\ldots,y_{ip})\) no componente \(h\) é o produto das marginais:

$$f_h(y_i;\,\psi_h) \;=\; \prod_{j=1}^{p} f_{NB}(y_{ij};\,\mu_{hj},\,\theta_{hj})$$
No código isso aparece como rowSums(dnbinom(...)) na escala log — a soma de log-densidades corresponde ao log do produto de densidades: 
# log f_h(y_i) = Σ_j log f_NB(y_ij ; μ_hj, θ_hj)
# rowSums devolve um vetor (n × 1), um valor por beneficiário i
log_f[, h] <- log(pi_h[h]) +
  rowSums(dnbinom(Y,
                  size = matrix(theta[h, ], n, p, byrow = TRUE),
                  mu   = matrix(mu[h, ],    n, p, byrow = TRUE),
                  log  = TRUE))
O argumento size do dnbinom do R corresponde ao \(\theta\) da NB₂ de Ng et al.: é o parâmetro de tamanho da NB (número de sucessos na formulação alternativa, ou equivalentemente o parâmetro de dispersão inverso).

4 · Log-verossimilhança dos dados completos

Log-lik Dados completos e incompletos (Ng et al. §1.3, p. 16–17)
O EM trata o problema de maximização introduzindo variáveis latentes \(Z_{ih} \in \{0,1\}\) que indicam se o beneficiário \(i\) pertence ao componente \(h\). Os dados completos são \((Y, Z)\); os dados observados (incompletos) são apenas \(Y\).

A log-verossimilhança dos dados completos factoriza como:

$$\ell_c(\Psi) \;=\; \sum_{i=1}^{n}\sum_{h=1}^{g} Z_{ih} \!\left[\log \pi_h + \sum_{j=1}^{p} \log f_{NB}(y_{ij};\,\mu_{hj},\theta_{hj})\right]$$
"The EM algorithm treats the allocation variables \(z_{ih}\) as missing data and iterates between the E-step, which computes the conditional expectation of \(\ell_c\) given \(y\), and the M-step, which maximizes this expectation." Ng, Xiang & Yau (2019) — §1.3, p. 16–17

A log-verossimilhança dos dados observados — monitorada para verificar convergência — é:

$$\ell(\Psi) \;=\; \sum_{i=1}^{n} \log \sum_{h=1}^{g} \pi_h\, \prod_{j=1}^{p} f_{NB}(y_{ij};\,\mu_{hj},\theta_{hj})$$

No código, log_norm contém \(\log f(y_i;\Psi)\) para cada \(i\), e loglik = sum(log_norm) é \(\ell(\hat{\Psi})\): 

# log_f[i,h] = log π_h + Σ_j log f_NB(y_ij; μ_hj, θ_hj)  ← log do integrando
# log_norm[i] = log Σ_h exp(log_f[i,h])                   ← log f(y_i; Ψ)
# loglik      = Σ_i log_norm[i]                            ← ℓ(Ψ)
loglik <- sum(log_norm)

5 · Inicialização: K-means + reinícios aleatórios

Init Estratégia de inicialização (Ng et al. §1.3.3, p. 18–19)
O EM pode convergir para máximos locais. Duas estratégias reduzem esse risco: (1) usar K-means como chute inicial informado — ele já fornece uma partição razoável dos dados; (2) repetir com reinícios aleatórios e reter o melhor.
"K-means clustering on a suitable transformation of the observed data provides component-membership-based starting values that are usually superior to random initializations for mixture models." Ng, Xiang & Yau (2019) — §1.3.3, p. 18–19

A transformação log1p(Y) estabiliza a escala das contagens (aproxima simetria) sem perder zeros, tornando a distância Euclidiana do K-means mais significativa:

$$\tilde{y}_{ij} = \log(1 + y_{ij})$$ 
for (init in seq_len(n_init)) {

  if (init == 1) {
    # ── Inicialização 1: K-means em log1p(Y)  ──────────────────────────────
    # Recomendado: Ng §1.3.3 p.18-19. nstart=10 para robustez do K-means.
    set.seed(seed)
    km  <- kmeans(log1p(Y), centers = g, nstart = 10, iter.max = 100)

    # Converte clusters em τ "mole": 0,98 no cluster atribuído, 0,02/(g-1) nos demais
    # Evita τ ∈ {0,1} que tranca θ no M-step
    tau <- matrix(0.02 / max(g - 1, 1), n, g)
    for (i in seq_len(n)) tau[i, km$cluster[i]] <- 0.98

  } else {
    # ── Inicializações 2..n_init: τ aleatório ──────────────────────────────
    # Perturbação de U(0,1)+0,01 evita τ exatamente zero
    set.seed(seed + init)
    tau <- matrix(runif(n * g) + 0.01, n, g)
  }

  # Normaliza τ para que Σ_h τ_ih = 1 (restrição de probabilidade)
  tau <- tau / rowSums(tau)

  # θ inicializado em 2 (sobredispersão moderada) para todos os componentes
  theta <- matrix(2, g, p)
O valor inicial theta = 2 corresponde a \(\theta = 2\), que implica \(\mathrm{Var}[Y] = \mu + \mu^2/2\) — sobredispersão moderada, nem muito rígida (Poisson, \(\theta \to \infty\)) nem muito difusa (\(\theta \to 0\)).

6 · Passo E — probabilidades posteriores

E-step Esperança condicional de Z (Ng et al. §1.3.2, p. 17)
No Passo E substituímos \(Z_{ih}\) pela sua esperança condicional dado \(Y = y\) e os parâmetros correntes \(\hat{\Psi}^{(t)}\). Essa esperança é a probabilidade posterior de pertencimento ao componente \(h\):
$$\hat{\tau}_{ih}^{(t)} \;=\; \frac{\pi_h^{(t)}\; \prod_{j=1}^{p} f_{NB}\!\left(y_{ij};\,\mu_{hj}^{(t)},\,\theta_{hj}^{(t)}\right)} {\displaystyle\sum_{k=1}^{g} \pi_k^{(t)}\; \prod_{j=1}^{p} f_{NB}\!\left(y_{ij};\,\mu_{kj}^{(t)},\,\theta_{kj}^{(t)}\right)}$$
"The E-step computes \(\tau_{ih} = E_\Psi[Z_{ih} \mid y_i]\), the posterior probability that the \(i\)th observation belongs to the \(h\)th component." Ng, Xiang & Yau (2019) — §1.3.2, p. 17

Estabilidade numérica: log-sum-exp

Computar as densidades diretamente na escala original pode levar a underflow numérico quando \(p\) é grande (produto de números muito pequenos). O truque log-sum-exp faz o cálculo inteiramente na escala logarítmica:
$$\log\!\left(\sum_{k=1}^{g} \exp(a_{ik})\right) \;=\; a_i^* + \log\!\left(\sum_{k=1}^{g} \exp(a_{ik} - a_i^*)\right), \quad a_i^* = \max_k a_{ik}$$

onde \(a_{ih} = \log \pi_h + \sum_j \log f_{NB}(y_{ij};\mu_{hj},\theta_{hj})\) é o log do numerador de \(\hat{\tau}_{ih}\). 

# ── Passo E: calcular log_f[i,h] para cada componente h ───────────────────
log_f <- matrix(0, n, g)
for (h in seq_len(g)) {
  log_f[, h] <- log(pi_h[h]) +           # log π_h
    rowSums(dnbinom(Y,                         # Σ_j log f_NB(y_ij; μ_hj, θ_hj)
                    size = matrix(theta[h, ], n, p, byrow = TRUE),
                    mu   = matrix(mu[h, ],    n, p, byrow = TRUE),
                    log  = TRUE))
}

# ── Log-sum-exp: log_max[i] = max_h log_f[i,h] ────────────────────────────
log_max  <- apply(log_f, 1, max)               # a*_i = max_h a_ih
log_norm <- log(rowSums(exp(log_f - log_max))) + log_max
#           └───────────────────────────────┘         │
#           log Σ_h exp(a_ih - a*_i)         +        a*_i
# = log f(y_i; Ψ)  (denominador de τ̂_ih em log)

loglik   <- sum(log_norm)                        # ℓ(Ψ) = Σ_i log f(y_i; Ψ)

# ── τ̂_ih = exp(log_f[i,h] - log_norm[i]) ─────────────────────────────────
tau_new  <- exp(log_f - log_norm)
tau_new  <- tau_new / rowSums(tau_new)          # re-normaliza (segurança numérica)

7 · Passo M — proporções π

M-step π Atualização das proporções de mistura (Ng et al. §1.3.2, p. 17)
Maximizar \(Q(\Psi;\hat{\Psi}^{(t)})\) em relação a \(\pi_h\) sujeito a \(\sum_h \pi_h = 1\) (via Lagrange) fornece a atualização fechada:
$$\hat{\pi}_h^{(t+1)} = \frac{\hat{n}_h}{n}, \qquad \hat{n}_h = \sum_{i=1}^{n} \hat{\tau}_{ih}^{(t)}$$
"The M-step update for the mixing proportions is \(\hat{\pi}_h = \hat{n}_h / n\), where \(\hat{n}_h = \sum_i \tau_{ih}\) is the effective number of observations in component \(h\)." Ng, Xiang & Yau (2019) — §1.3.2, p. 17 
# π̂_h = n̂_h / n  (colMeans(τ) = (Σ_i τ_ih)/n)
pi_h <- colMeans(tau)        # vetor g × 1
n_h  <- colSums(tau)         # n̂_h = Σ_i τ_ih  (usado no M-step de μ)
colMeans(tau) é exatamente \(\hat{n}_h / n\). O vetor n_h (colSums(tau)) armazena \(\hat{n}_h\) para reutilização na atualização de \(\mu\).

8 · Passo M — médias μ

M-step μ Médias ponderadas por componente (Ng et al. §4.4, p. 82)
Para a NB₂, a derivada parcial de \(Q\) em relação a \(\mu_{hj}\) se anula em um estimador de média ponderada pelas probabilidades posteriores — análogo ao M-step da mistura Poisson, com a mesma forma fechada:
$$\hat{\mu}_{hj}^{(t+1)} \;=\; \frac{\displaystyle\sum_{i=1}^{n} \hat{\tau}_{ih}^{(t)}\, y_{ij}} {\displaystyle\sum_{i=1}^{n} \hat{\tau}_{ih}^{(t)}} \;=\; \frac{\sum_i \hat{\tau}_{ih}\, y_{ij}}{\hat{n}_h}$$
"For the NB mixture, the M-step update for \(\mu_{hj}\) is the posterior-weighted sample mean \(\hat{\mu}_{hj} = \sum_i \tau_{ih} y_{ij} / \sum_i \tau_{ih}\)." Ng, Xiang & Yau (2019) — §4.4, p. 82 
# μ̂_hj = (τ^T Y)_hj / n̂_h
# t(tau) %*% Y  →  g × p  (produto matricial ponderado)
# / n_h         →  divisão elemento a elemento por coluna
mu <- pmax(t(tau) %*% Y / n_h, 1e-6)
#            └─── t(tau) é g×n; Y é n×p ──────────┘
# pmax(..., 1e-6) garante μ > 0 (log-link sempre válido)
O produto t(tau) %*% Y calcula em uma operação matricial \(\sum_i \hat{\tau}_{ih} y_{ij}\) para todos os \(h,j\) simultaneamente, evitando laços explícitos e sendo computacionalmente eficiente.

9 · Passo M — dispersão θ

M-step θ Otimização numérica do parâmetro de dispersão (Ng et al. §4.4, p. 82–83)
Diferentemente de \(\mu\) e \(\pi\), o parâmetro de dispersão \(\theta_{hj}\) não admite fórmula fechada. Deve-se maximizar numericamente a verossimilhança ponderada parcial:
$$\hat{\theta}_{hj}^{(t+1)} \;=\; \arg\max_{\theta > 0}\; \sum_{i=1}^{n} \hat{\tau}_{ih}^{(t)}\, \log f_{NB}\!\left(y_{ij};\,\hat{\mu}_{hj}^{(t+1)},\,\theta\right)$$
"The dispersion parameter \(\theta_{hj}\) does not have a closed-form M-step update; it is obtained by one-dimensional numerical optimization of the weighted log-likelihood with respect to \(\theta_{hj}\) for fixed \(\mu_{hj}\)." Ng, Xiang & Yau (2019) — §4.4, p. 82–83

Implementação: log-espaço para garantir θ > 0

Otimizar \(\log\theta\) em vez de \(\theta\) diretamente garante \(\theta > 0\) sem precisar de restrições de caixa; o optimize busca no intervalo \((\log\theta_{\min},\, \log\theta_{\max}) = (-4, 8)\), correspondendo a \(\theta \in (e^{-4}, e^{8}) \approx (0.018,\, 2981)\). 
# .update_theta: maximização unidimensional de θ_hj em log-espaço
# Argumentos:
#   y_j    = Y[,j]       vetor n × 1 de contagens da variável j
#   tau_h  = tau[,h]     vetor n × 1 de posteriores do componente h
#   mu_hj  = mu[h,j]     média estimada no M-step anterior
.update_theta <- function(y_j, tau_h, mu_hj) {

  neg_ll <- function(lt) {
    th <- exp(lt)                                          # θ = exp(log θ) > 0
    -sum(tau_h * dnbinom(y_j, size = th, mu = mu_hj, log = TRUE))
    # ─────────────────────────────────────────────────────────────────────
    # - Σ_i τ_ih · log f_NB(y_ij; μ_hj, θ)  ← neg. lik. ponderada parcial
  }

  # optimize minimiza neg_ll em lt ∈ (-4, 8)  → θ ∈ (0.018, 2981)
  exp(tryCatch(optimize(neg_ll, interval = c(-4, 8))$minimum,
               error = function(e) 0))    # fallback exp(0)=1 se numérico falhar
}

# Chamada dentro do loop EM (para cada h e j):
for (h in seq_len(g))
  for (j in seq_len(p))
    theta[h, j] <- .update_theta(Y[, j], tau[, h], mu[h, j])

# Guarda θ ≥ 0.05: dispersão muito baixa (θ→0) causaria instabilidade
theta <- pmax(theta, 0.05)
O tryCatch absorve falhas numéricas raras (e.g., gradiente indefinido quando todos os \(y_{ij} = 0\)), retornando exp(0) = 1 como valor seguro. O pmax(theta, 0.05) bloqueia valores extremamente próximos de zero que tornariam \(\mathrm{Var}[Y] \approx \infty\).

10 · Critério de convergência e múltiplos reinícios

Convergência Monotonicidade e critério de parada (Ng et al. §1.3.2, p. 17–18)
O EM garante que \(\ell(\Psi^{(t+1)}) \geq \ell(\Psi^{(t)})\) em cada iteração (propriedade de ascensão). A convergência é declarada quando o incremento fica abaixo de uma tolerância \(\varepsilon\):
$$\left|\ell(\Psi^{(t)}) - \ell(\Psi^{(t-1)})\right| < \varepsilon, \qquad \varepsilon = 10^{-5}$$ 
# Critério de convergência: |ℓ^(t) - ℓ^(t-1)| < tol  (tol = 1e-5)
if (iter > 1 && abs(loglik - loglik_old) < tol) break
loglik_old <- loglik
tau        <- tau_new    # avança τ para a próxima iteração

Seleção do melhor reinício

Após cada reinício (inicialização diferente), o valor final de \(\ell(\hat{\Psi})\) é comparado ao melhor encontrado até então. Apenas o modelo com maior log-verossimilhança é retornado: 

# Atualiza best_fit somente se este reinício superou o anterior
if (loglik > best_loglik) {
  best_loglik <- loglik
  k_params    <- 2L * g * p + (g - 1L)    # d(g) = 2gp + (g-1)  ← ver §BIC
  best_fit    <- list(
    g = g, pi = pi_h, mu = mu, theta = theta, tau = tau_new,
    loglik = loglik, n_iter = iter,
    classification = apply(tau_new, 1, which.max),
    BIC = -2 * loglik + k_params * log(n),
    AIC = -2 * loglik + 2 * k_params
  )
}
O campo classification (apply(tau_new, 1, which.max)) implementa a regra MAP (maximum a posteriori): cada beneficiário é alocado ao componente com maior probabilidade posterior — o classificador de Bayes ótimo sob perda 0-1.

11 · Seleção de g por BIC

BIC Critério de seleção do número de componentes (Ng et al. §1.2.3, p. 12)
O BIC (Bayesian Information Criterion) penaliza a log-verossimilhança maximizada pelo número de parâmetros, equilibrando ajuste e parcimônia:
$$BIC(g) \;=\; -2\,\hat{\ell}(\hat{\Psi}) + d(g)\,\log n$$

A contagem de parâmetros independentes para um modelo com \(g\) componentes, \(p\) variáveis e NB₂ por variável é:

$$d(g) \;=\; \underbrace{gp}_{\mu_{hj}} + \underbrace{gp}_{\theta_{hj}} + \underbrace{(g-1)}_{\pi_h} \;=\; 2gp + (g-1)$$
"The BIC selects the model with the fewest parameters that still provides a good fit; for a \(g\)-component mixture with \(p\)-dimensional NB components, the number of free parameters is \(d(g) = 2gp + (g-1)\)." Ng, Xiang & Yau (2019) — §1.2.3, p. 12 
# d(g) = 2·g·p + (g-1):  2gp parâmetros de (μ,θ) + (g-1) proporções livres
k_params <- 2L * g * p + (g - 1L)

# BIC = -2 ℓ̂(Ψ̂) + d(g) · log n
BIC = -2 * loglik + k_params * log(n)

# Seleção: g* = argmin_g BIC(g)  (loop externo em comparar_variaveis.R)
M1_G <- which.min(M1_bic)    # índice do g ótimo
M1_opt <- M1_fits[[M1_G]]   # fit com menor BIC

O BIC é avaliado para \(g = 1, 2, \ldots, G_{\max}\) (padrão: G_MAX = 6). O modelo selecionado é o com menor \(BIC(g)\).

12 · Pareamento ótimo: algoritmo húngaro

Avaliação Pareamento de rótulos para métricas de recuperação
Modelos de mistura produzem rótulos de componente arbitrários (permutação do verdadeiro). Para comparar classificação estimada com rótulos reais é necessário encontrar a permutação ótima que maximiza a concordância — problema de atribuição resolvido pelo algoritmo húngaro.
$$\hat{\sigma} \;=\; \arg\max_{\sigma \in S_g} \sum_{i=1}^{n} \mathbf{1}\!\left[\hat{z}_i = \sigma(z_i)\right]$$ 
# parear_otimo: usa clue::solve_LSAP (Linear Sum Assignment Problem)
parear_otimo <- function(real, estimado) {
  g    <- max(max(real), max(estimado))
  custo <- matrix(0, g, g)

  # Matriz de concordância: custo[i,j] = -#{real==i & estimado==j}
  for (i in seq_len(max(real)))
    for (j in seq_len(max(estimado)))
      custo[i, j] <- -sum(real == i & estimado == j)

  # solve_LSAP encontra a permutação que minimiza o custo total
  # (equivalente a maximizar concordância)
  sol <- clue::solve_LSAP(custo + abs(min(custo)))
  order(as.integer(sol))[estimado]    # rótulos remapeados
}

Com os rótulos pareados, calculam-se as métricas de recuperação:

Métrica Fórmula / Interpretação Intervalo
ARI
Adjusted Rand Index		 Concordância de pares ajustada pelo acaso [-1, 1] · ótimo = 1
NMI
Normalized Mutual Info		 Informação mútua entre classificações, normalizada [0, 1] · ótimo = 1
Acurácia \(n^{-1}\sum_i \mathbf{1}[\hat{z}_i^{(\sigma)} = z_i]\) após pareamento [0, 1] · ótimo = 1
Pureza Média da proporção do grupo dominante por cluster [0, 1] · ótimo = 1

13 · Código completo anotado

Abaixo reproduzimos o núcleo do algoritmo EM — as funções .update_theta e nb_mix_em — com anotações inline referenciando as seções acima: 

# =============================================================================
# Função auxiliar: otimização unidimensional de θ_hj
# Fundamentação: Ng §4.4 p.82-83 (M-step θ sem forma fechada)
# =============================================================================
.update_theta <- function(y_j, tau_h, mu_hj) {
  neg_ll <- function(lt) {
    th <- exp(lt)                                          # lt = log θ → θ > 0
    -sum(tau_h * dnbinom(y_j, size = th, mu = mu_hj, log = TRUE))
  }
  exp(tryCatch(optimize(neg_ll, interval = c(-4, 8))$minimum,
               error = function(e) 0))
}


# =============================================================================
# Algoritmo EM principal: mistura de g NB multivariadas
# =============================================================================
nb_mix_em <- function(Y, g,
                      max_iter = MAX_ITER_NB, tol = TOL_NB,
                      n_init = N_INIT_NB, seed = SEED) {
  n <- nrow(Y); p <- ncol(Y)
  Y <- as.matrix(Y)
  best_loglik <- -Inf; best_fit <- NULL

  for (init in seq_len(n_init)) {
    # ── INICIALIZAÇÃO (§5) ────────────────────────────────────────────────────
    if (init == 1) {
      set.seed(seed)
      km  <- kmeans(log1p(Y), centers = g, nstart = 10, iter.max = 100)
      tau <- matrix(0.02 / max(g - 1, 1), n, g)
      for (i in seq_len(n)) tau[i, km$cluster[i]] <- 0.98
    } else {
      set.seed(seed + init)
      tau <- matrix(runif(n * g) + 0.01, n, g)
    }
    tau   <- tau / rowSums(tau)
    theta <- matrix(2, g, p)
    loglik_old <- -Inf

    for (iter in seq_len(max_iter)) {
      # ── PASSO M: π e μ (§7, §8) ───────────────────────────────────────────
      pi_h <- colMeans(tau)                              # π̂_h = n̂_h / n
      n_h  <- colSums(tau)                               # n̂_h = Σ_i τ_ih
      mu   <- pmax(t(tau) %*% Y / n_h, 1e-6)           # μ̂_hj (g×p)

      # ── PASSO M: θ (§9) — otimização unidimensional por (h,j) ─────────────
      for (h in seq_len(g))
        for (j in seq_len(p))
          theta[h, j] <- .update_theta(Y[, j], tau[, h], mu[h, j])
      theta <- pmax(theta, 0.05)                         # guarda θ ≥ 0.05

      # ── PASSO E: log_f e log-sum-exp (§6) ────────────────────────────────
      log_f <- matrix(0, n, g)
      for (h in seq_len(g)) {
        log_f[, h] <- log(pi_h[h]) +
          rowSums(dnbinom(Y,
                          size = matrix(theta[h, ], n, p, byrow = TRUE),
                          mu   = matrix(mu[h, ],    n, p, byrow = TRUE),
                          log  = TRUE))
      }
      log_max  <- apply(log_f, 1, max)                   # a*_i
      log_norm <- log(rowSums(exp(log_f - log_max))) + log_max
      loglik   <- sum(log_norm)                          # ℓ(Ψ)
      tau_new  <- exp(log_f - log_norm)
      tau_new  <- tau_new / rowSums(tau_new)             # τ̂_ih

      # ── CONVERGÊNCIA (§10) ────────────────────────────────────────────────
      if (iter > 1 && abs(loglik - loglik_old) < tol) break
      loglik_old <- loglik; tau <- tau_new
    }

    # ── SELECIONA MELHOR REINÍCIO (§10) + CALCULA BIC (§11) ───────────────
    if (loglik > best_loglik) {
      best_loglik <- loglik
      k_params    <- 2L * g * p + (g - 1L)               # d(g) = 2gp + (g-1)
      best_fit    <- list(
        g = g, pi = pi_h, mu = mu, theta = theta, tau = tau_new,
        loglik = loglik, n_iter = iter,
        classification = apply(tau_new, 1, which.max),   # regra MAP
        BIC = -2 * loglik + k_params * log(n),
        AIC = -2 * loglik + 2 * k_params
      )
    }
  }
  best_fit
}
