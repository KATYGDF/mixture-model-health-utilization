1 · Motivação

Dados reais de utilização de planos de saúde não estão disponíveis publicamente com a granularidade necessária para estudar heterogeneidade latente. A alternativa adotada é a simulação por processo generativo: especificamos explicitamente o modelo probabilístico que acreditamos descrever a realidade e geramos uma amostra a partir dele. Esse procedimento tem duas vantagens metodológicas:

  1. O grupo latente verdadeiro de cada beneficiário é conhecido — o que permite avaliar métricas de recuperação como ARI e acurácia, impossíveis de calcular com dados reais sem gabarito. Gonzales et al. (2023) identificam simulação e pesquisa preditiva como o principal caso de uso de dados sintéticos em saúde, destacando que permitem expandir amostras e testar algoritmos em condições controladas. Tucker et al. (2020) complementam:
"One possible solution to this problem is the use of synthetic data as an alternative to assist in the rapid development and validation of new tools. This data must capture all of the correct (potentially non-linear and multivariate) dependencies and distributions that are apparent in the real data sets." Tucker et al. (2020, p. 1)
  1. Os parâmetros podem ser calibrados a indicadores públicos do Painel de Indicadores da ANS, tornando a simulação realista. Avanzi et al. (2021) fundamentam essa abordagem ao argumentar que modelos propostos devem sempre ser testados contra dados sintéticos, pois
"the features contained in the synthetic data set will be known, by construction. One is then able to test the extent to which the proposed model is successful in identifying them." Avanzi et al. (2021, p. 296)

Os dados são inteiramente sintéticos. Nenhum beneficiário real está representado.

2 · Mistura finita de distribuições

Deb & Trivedi (1997) foram os primeiros a aplicar modelos de mistura finita em econometria da saúde, motivados pela insuficiência dos modelos padrão para capturar heterogeneidade latente na utilização de serviços médicos:

"The finite mixture model allows for additional population heterogeneity but avoids the sharp dichotomy between the population of 'users' and 'non-users'. [...] the underlying unobserved heterogeneity which splits the population into latent classes is assumed to be based on the person's latent long-term health status." Deb & Trivedi (1997, p. 314)

A fundamentação teórica completa da estrutura de mistura finita — incluindo condições de identificabilidade e propriedades assintóticas — encontra-se em McLachlan & Peel (2000).

Seja \(Y_i\) o vetor de contagens do beneficiário \(i\) (consultas, pronto-socorro, exames, terapias, internações). A hipótese central é que a população é composta por \(G = 4\) subpopulações — grupos latentes — com perfis de utilização qualitativamente distintos. A densidade marginal de \(Y_i\) é uma mistura:

$$f(y_i;\,\Psi) \;=\; \sum_{h=1}^{G} \pi_h \cdot f_h(y_i;\,\psi_h)$$

onde \(\pi_h > 0\) é a proporção do grupo \(h\), com \(\sum_{h=1}^{G} \pi_h = 1\), e \(f_h(\,\cdot\,;\psi_h)\) é a densidade do grupo \(h\) com parâmetros \(\psi_h\). O conjunto completo de parâmetros é \(\Psi = (\pi_1, \ldots, \pi_G,\, \psi_1, \ldots, \psi_G)\).

A pertença de grupo é introduzida por meio de uma variável latente \(Z_i \in \{1, \ldots, G\}\) com distribuição multinomial:

$$P(Z_i = h) = \pi_h, \qquad h = 1, \ldots, G$$

Dado \(Z_i = h\), as componentes de \(Y_i\) são condicionalmente independentes e cada componente segue uma Binomial Negativa (ou ZINB para terapias e internações):

$$Y_{ij} \mid Z_i = h \;\sim\; \mathrm{NB}(\mu_{ihj},\, \theta_{hj})$$

Algoritmo EM

A estimação da mistura por máxima verossimilhança é realizada pelo Algoritmo EM (Expectation-Maximization, Redner & Walker, 1984), que alterna dois passos até convergência. Para misturas de GLMs com log-link, Jansen (1993) deriva o Passo M analiticamente. Eckardt et al. (2017) descrevem a implementação prática:

"Because of the analytical limitations of solving the maximum likelihood equation [...] we implemented an expectation maximization (EM) algorithm for estimation purpose [...]. Thus, an iterative procedure was chosen, in which the component membership was taken as missing value." Eckardt et al. (2017, p. 588)

Passo E — calcular as probabilidades de pertença a posteriori (responsabilidades) pelo teorema de Bayes:

$$\tau_{ih}^{(t)} \;=\; P\!\left(Z_i = h \mid y_i,\,\Psi^{(t)}\right) \;=\; \frac{\pi_h^{(t)}\, f_h\!\left(y_i;\,\psi_h^{(t)}\right)} {\displaystyle\sum_{k=1}^{G} \pi_k^{(t)}\, f_k\!\left(y_i;\,\psi_k^{(t)}\right)}$$

Passo M — maximizar a log-verossimilhança completa esperada:

$$\pi_h^{(t+1)} = \frac{1}{n}\sum_{i=1}^{n} \tau_{ih}^{(t)} \qquad \psi_h^{(t+1)} = \underset{\psi}{\arg\max} \sum_{i=1}^{n} \tau_{ih}^{(t)} \log f_h(y_i;\,\psi)$$

A convergência é verificada pelo critério \(\lvert \ell^{(t+1)} - \ell^{(t)} \rvert < \varepsilon\) com tolerância \(\varepsilon = 10^{-5}\). O número de grupos \(G\) é selecionado pelo BIC:

$$\mathrm{BIC} = -2\,\hat{\ell} + k \cdot \log n$$

onde \(k\) é o número de parâmetros livres. Menor BIC indica melhor equilíbrio entre ajuste e parcimônia. A escolha de valores iniciais adequados é crítica para a convergência do EM, dado o risco de mínimos locais (Karlis & Xekalaki, 2003); por isso são utilizadas múltiplas inicializações, com a primeira baseada em K-means sobre \(\log(1 + Y)\).

3 · Os quatro grupos latentes

A estrutura de grupos latentes tem precedente direto na literatura. Deb & Holmes (2000) identificaram classes distintas — os worried well e os severamente enfermos — e mostraram que modelos padrão subestimam sistematicamente os usuários de alta intensidade:

"Unlike standard specifications, the FMM has the ability to distinguish between distinct classes of users of behavioural health care (e.g. the 'worried well' and the severely mentally ill). [...] Standard models consistently fail to accurately estimate the resource consumption needs of this group [high intensity users]." Deb & Holmes (2000, pp. 475–477)

Lourenço & Ferreira (2005) reforçam que ignorar a heterogeneidade não observada em modelos de utilização "leads to inconsistent parameters estimates" (p. 945). No contexto brasileiro, o Mapa Assistencial da ANS (IESS, 2024) documenta crescimento diferenciado de 7,4 % nos beneficiários entre 2019 e 2023, com padrões heterogêneos por faixa etária — base empírica para as proporções e perfis adotados. As proporções populacionais adotadas são:

4 · Distribuição Binomial Negativa (NB₂)

Dados de utilização de serviços de saúde são contagens com sobredispersão — variância muito maior que a média — o que inviabiliza a distribuição de Poisson. Wagner et al. (2015) alertam para as consequências de ignorar esse fenômeno:

"Overdispersion is common and occurs when patient variability results in data variance which is greater than that assumed by the distribution. Unaccounted for, overdispersion will result in incorrect predictions and will tend to overestimate standard errors increasing the type I error rate [...] The overdispersion parameter was significant in all models, indicating that a simpler Poisson or binomial model for these data would be inadequate." Wagner et al. (2015, p. 490)

Abu Bakar et al. (2022) confirmam, ao comparar sistematicamente Poisson, NB, ZIP e ZINB para utilização ambulatorial: "the NB model is equipped for a parameter to account for overdispersion. It is deemed to be a better model for an overdispersion variance to mean" (p. 3). Lee et al. (2023) complementam com critérios práticos de seleção de modelo. A parametrização NB₂ (Binomial Negativa de média) adotada tem função de massa de probabilidade:

$$P(Y = y;\,\mu,\theta) \;=\; \frac{\Gamma(y+\theta)}{\Gamma(\theta)\,y!} \left(\frac{\theta}{\theta+\mu}\right)^{\!\theta} \left(\frac{\mu}{\theta+\mu}\right)^{\!y}, \quad y \in \{0,1,2,\ldots\}$$

Os momentos são:

$$\mathrm{E}[Y] = \mu \qquad \mathrm{Var}[Y] = \mu + \frac{\mu^2}{\theta}$$

Quando \(\theta \to \infty\) a variância tende a \(\mu\) e a NB₂ converge para a Poisson. Valores pequenos de \(\theta\) indicam alta sobredispersão. Na simulação:

5 · ZINB — zeros estruturais

Terapias (~80 % de zeros) e internações (~94 % de zeros) apresentam excesso de zeros impossível de acomodar pela NB₂ sozinha. Abu Bakar et al. (2022) distinguem dois tipos de zeros em dados de utilização: "true zero reflects non-users because they did not get sick during the reference period, while frequency zero reflects individuals who fell sick during the reference period but did not seek care" (p. 3). Wagner et al. (2015) descrevem o mecanismo gerador no contexto de dados de saúde:

"These zero counts are thought to occur for two reasons: (i) the subject does not use that particular drug (true zero) or (ii) the subject does use the drug but did not report use during the time of observation [...] a zero-inflated (ZI) model is attractive in assuming that zero counts result from a mixture of two distributions, one where subjects always produce zero counts (non-users) and one where subjects who do use the substance produce zero counts by chance." Wagner et al. (2015, p. 491)

Preisser et al. (2016) e Lewsey & Thomson (2004) documentam a mesma estrutura em dados dentários e longitudinais de saúde. O modelo ZINB mistura um componente degenerado em zero com a NB:

$$P(Y = 0) \;=\; \omega + (1-\omega)\,P_{\mathrm{NB}}(Y=0;\,\mu,\theta)$$ $$P(Y = y) \;=\; (1-\omega)\,P_{\mathrm{NB}}(Y=y;\,\mu,\theta), \quad y \geq 1$$

onde \(\omega \in [0,1)\) é a probabilidade de pertencer ao componente de zeros estruturais. Os valores por grupo são:

Desfecho \(\omega\) (\(G_1\)) \(\omega\) (\(G_2\)) \(\omega\) (\(G_3\)) \(\omega\) (\(G_4\))
Terapias 0,850,450,700,35
Internações 0,970,940,550,75

\(G_3\) tem a menor probabilidade de zero em internações (0,55), refletindo o perfil agudo/hospitalar. Para terapias, é \(G_4\) com menor \(\omega\) (0,35), consistente com uso intensivo e atípico de serviços especializados.

6 · Log-link e covariáveis

A média condicional de cada desfecho depende do grupo latente e das covariáveis observadas via modelo de regressão com log-link. Essa estrutura — em que cada componente da mistura é um GLM com log-link — foi formalizada por Jansen (1993) e aplicada ao contexto de custos assistenciais com covariáveis socioeconômicas por Eckardt et al. (2017):

$$\log(\mu_{ihj}) \;=\; \alpha_{hj} + \mathbf{X}_i^\top \boldsymbol{\beta}_{hj}$$

onde \(\alpha_{hj}\) é o intercepto do grupo \(h\) para o desfecho \(j\), \(\mathbf{X}_i\) é o vetor de covariáveis do beneficiário \(i\), e \(\boldsymbol{\beta}_{hj}\) são os coeficientes grupo-específicos. O log-link garante \(\mu_{ihj} > 0\) para qualquer valor das covariáveis. As covariáveis incluídas são:

CovariávelDescriçãoDistribuição simulada
\(\texttt{idade10}\) Idade / 10 (contínua) Misto uniforme: 20 % (0–17), 55 % (18–59), 25 % (60–85)
\(\texttt{sexo}\) 1 = feminino, 0 = masculino Bernoulli(0,55)
\(\texttt{tempo}\) Tempo no plano (anos) Uniforme(1/12, 10)
\(\texttt{copart}\) 1 = com coparticipação Bernoulli(0,40)
\(\texttt{regiao2},\,\texttt{regiao3}\) Indicadores de região (vs. região 1) Multinomial(0,40; 0,35; 0,25)

7 · Coeficientes grupo-específicos

Os coeficientes \(\boldsymbol{\beta}_{hj}\) variam entre grupos, permitindo que o efeito de cada covariável seja calibrado ao perfil de cada subpopulação. Os valores foram definidos para refletir a heterogeneidade observada no uso de serviços de saúde suplementar no Brasil. Evidências empíricas do setor indicam crescimento diferenciado entre tipos de procedimentos e faixas etárias, sugerindo a existência de subpopulações com distintos padrões de utilização (IESS, 2024; Albuquerque et al., 2008; Vitorino et al., 2024). Nesse contexto, os parâmetros foram calibrados para capturar:

  • maior utilização associada ao envelhecimento (coeficiente positivo de \(\texttt{idade10}\) nos grupos \(G_1\)–\(G_3\));
  • redução do uso em função da coparticipação financeira (coeficiente negativo de \(\texttt{copart}\) em todos os grupos);
  • diferenças comportamentais entre grupos latentes, incluindo perfis de baixa utilização (\(G_1\)), alta necessidade clínica (\(G_2\), \(G_3\)) e padrões atípicos potencialmente associados a fraude (\(G_4\)).

8 · Beta-Binomial — meses de utilização

A variável \(\texttt{meses\_utilizacao}\) conta os meses com pelo menos um evento registrado. É limitada ao intervalo inteiro \([1, 12]\) e apresenta distribuição bimodal em U — padrão em que a maioria dos beneficiários usa o plano ou esporadicamente ou durante quase todos os meses. Liu et al. (2013) documentaram exatamente essa forma em dados de utilização de VA e caracterizaram a especificação adequada:

"Beta-binomial regression is a rarely used model in health services research that has the flexibility to fit various distributions, including bimodal and U-shaped distributions. [...] The beta-binomial model [...] is most useful in the case of extreme over-dispersion that cannot be addressed with a negative binomial model, such as U-shaped distributions with finite support bounded between 0 and 1." Liu et al. (2013, p. 1772)

Wagner et al. (2015) confirmam empiricamente a superioridade da Beta-Binomial sobre NB e ZINB para variáveis limitadas com distribuição em U: "the beta-binomial better captured the counts represented by the observed distribution" (p. 494). O modelo adotado é a distribuição Beta-Binomial em dois estágios:

$$p_i \;\sim\; \mathrm{Beta}(a_i,\, b_i)$$ $$M_i \mid p_i \;\sim\; \mathrm{Binomial}(12,\, p_i)$$

Os hiperparâmetros são definidos em função do volume total de eventos do beneficiário:

$$a_i = \max\!\left(0{,}5;\;\frac{\text{total\_eventos}_i}{6}\right) \qquad b_i = \max\!\left(0{,}5;\;\frac{36 - \text{total\_eventos}_i}{6}\right)$$

Beneficiários com muitos eventos têm \(a_i \gg b_i\) (uso todo o ano); com poucos eventos têm \(b_i \gg a_i\) (uso pontual).

Nota sobre a modelagem: A distribuição Beta-Binomial é distinta da NB₂ e, por isso, a mistura NB usada no script de análise a aproxima de forma imperfeita. Resultados para conjuntos que incluem meses_utilizacao devem ser interpretados com cautela.

9 · Gasto total

O gasto total é calculado como combinação linear das contagens por tipo de evento, multiplicada por um ruído log-normal que representa a variação de preços entre prestadores e regiões. Os custos médios unitários foram calibrados com referência em Andrade et al. (2013), que analisam indicadores de gastos desagregados por tipo de serviço no setor suplementar brasileiro, e em Avanzi et al. (2021), que adotam estrutura análoga em simuladores de sinistros individuais:

$$\text{gasto}_i \;=\; \bigl(120\,\text{cons}_i + 250\,\text{ps}_i + 80\,\text{ex}_i + 150\,\text{ter}_i + 3500\,\text{int}_i\bigr)\cdot \varepsilon_i, \qquad \varepsilon_i \sim \mathrm{LogNormal}(0,\,0{,}20)$$

10 · Calibração aos indicadores ANS

Os interceptos \(\alpha_{hj}\) foram ajustados iterativamente para que as médias marginais simuladas fiquem dentro de 10 % dos alvos do Painel de Indicadores da ANS. As séries de referência provêm do Mapa Assistencial publicado pelo IESS (2024), que registra a produção de 1,7 bilhão de procedimentos médico-hospitalares em 2023 — crescimento de 7,5 % em relação a 2022 e de 18,5 % em relação a 2019, com exames complementares crescendo 27,4 % no mesmo período (IESS, 2024, pp. 9–10). calc_mu <- function(alpha_vec, beta_mat, grupo, idade10, sexo,
                    tempo_anos, copart, regiao2, regiao3) {

  X <- cbind(idade10, sexo, tempo_anos, copart, regiao2, regiao3)
  colnames(X) <- c("idade10","sexo","tempo","copart","regiao2","regiao3")

  beta_ind <- beta_mat[grupo, ]
  eta <- alpha_vec[grupo] + rowSums(X * beta_ind)

  exp(eta)
}

rnbinom_mu <- function(mu, theta) {
  rnbinom(n = length(mu), size = theta, mu = mu)
}

rzinb_mu <- function(mu, theta, pzero) {
  zero_flag <- rbinom(length(mu), size = 1, prob = pzero)
  y_nb      <- rnbinom(n = length(mu), size = theta, mu = mu)
  ifelse(zero_flag == 1, 0, y_nb)
}

  mu    = mu_terapias,
  theta = theta_terapias,

  theta = theta_internacoes,
  pzero = pzero_internacoes[grupo_latente]
)

# --- Gasto total simulado -----------------------------------
# Custos medios por evento (R$):
# consulta = 120 | ps = 250 | exame = 80 | terapia = 150 | internacao = 3500
# Ruido log-normal com CV aproximado de 20%

gasto_base <- consultas   * 120 +
              ps          * 250 +
              exames      * 80  +
              terapias    * 150 +
              internacoes * 3500

ruido_gasto <- rlnorm(n, meanlog = 0, sdlog = 0.20)
gasto_total <- round(gasto_base * ruido_gasto, 2)

# --- meses_utilizacao: Beta-Binomial ------------------------
shape_a <- pmax(0.5, total_eventos / 6)
shape_b <- pmax(0.5, (36 - total_eventos) / 6)

p_mes <- rbeta(n, shape1 = shape_a, shape2 = shape_b)
meses_utilizacao <- rbinom(n, size = 12, prob = p_mes)
meses_utilizacao <- pmax(1L, meses_utilizacao)

# --- prestadores_distintos com cap inteiro ------------------
prest_raw <- pmax(1L, rpois(n, lambda = pmax(1, total_eventos / 5)))
cap_prest <- as.integer(quantile(prest_raw, 0.99))
prestadores_distintos <- pmin(prest_raw, cap_prest)

# --- Flags --------------------------------------------------

  total_eventos >= quantile(total_eventos, 0.95),
  1L, 0L
)
